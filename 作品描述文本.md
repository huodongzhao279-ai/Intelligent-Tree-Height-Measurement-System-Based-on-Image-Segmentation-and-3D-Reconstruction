# 作品描述文本

## 作品简介

本作品名称为“基于图像分割与三维重建的树木高度智能测量系统”，面向校园树木普查、园林养护、林业资源调查与智慧农林监测等应用场景，旨在解决传统树木高度测量效率低、设备依赖强、使用门槛高等问题。系统综合利用图像分割、深度信息处理、单目三维重建、点云筛选与几何测量等技术，实现树木目标识别、三维点云生成以及树木高度自动估计。

本系统支持两种工作模式。第一种模式基于移动设备深度数据进行点云构建与高度测量；第二种模式基于单目 RGB 图像进行三维重建，在无深度传感器条件下同样可以完成树木高度估计，从而提升了系统对普通手机和相机设备的兼容性。系统后端基于 Flask 构建，支持原始协议上传、简化接口调用、可视化结果返回及点云文件导出。

从整体流程上看，系统首先接收树木图像和用户提示点，随后通过分割模型生成树木区域掩码，再结合深度信息或三维重建模型生成场景点云，并依据树木掩码提取目标点云。最后通过离群点去除和 AABB 包围盒分析计算树木高度，输出带有高度标注的可视化结果和点云文件。本作品强调“从图像输入到测量结果输出”的完整工程链路，兼顾算法实现与实际应用价值。

## 开源代码与组件使用情况说明

本作品在自主完成系统设计、功能整合、接口开发、任务适配与工程实现的基础上，使用了若干开源组件与第三方依赖，以提升开发效率和算法能力。相关情况说明如下：

### 1. 使用的主要开源组件

- `Flask`：用于构建后端接口服务。
- `PyTorch`：用于深度学习模型加载与推理。
- `OpenCV`：用于图像处理、阈值分割、轮廓提取和可视化绘制。
- `Open3D`：用于点云处理、点云去噪和点云文件导出。
- `Pillow`：用于图像读取、格式转换和结果保存。
- `NumPy`：用于数值计算和数据处理。
- `segment-anything`：用于根据提示点生成树木目标分割掩码。
- `SLAM3R` 集成代码：用于单目 RGB 图像场景下的三维重建与点云生成。

### 2. 开源组件在作品中的用途

- 分割模型主要用于树木区域提取，为后续点云筛选提供目标掩码。
- 深度学习框架主要用于模型推理与 GPU/CPU 运行支持。
- 图像处理与点云处理组件主要用于生成测量所需的中间结果和最终结果。
- Web 框架主要用于对外提供接口服务和结果下载能力。

### 3. 团队自主完成的工作

本作品的系统架构设计、树木测量任务建模、深度模式与单目重建模式的功能整合、输入输出协议设计、结果可视化逻辑、点云筛选与高度测量流程、接口安全校验以及整体工程实现，均由团队自主完成。团队的工作重点不在于重新训练底层通用模型，而在于针对树木高度测量这一具体任务进行算法组合、流程适配、工程整合与应用落地。

### 4. 说明

正式提交时，建议补充每个开源组件的版本号、开源协议以及项目中的具体使用位置，形成更完整的“开源组件清单”。

## 作品安装说明

### 1. 运行环境

- 操作系统：Windows 10/11 或 Linux
- Python 版本：建议 Python 3.11
- 显卡环境：建议具备 NVIDIA GPU，以获得更好的推理速度
- 运行方式：后端接口服务

### 2. 项目依赖安装

深度模式可参考项目中的 `requirements.txt`，单目三维重建模式可参考 `requirements_slam3r.txt`。

建议安装步骤如下：

```bash
conda create -n treeheight_slam3r python=3.11
conda activate treeheight_slam3r
pip install torch==2.5.0 torchvision==0.20.0 --index-url https://download.pytorch.org/whl/cu118
pip install -r requirements_slam3r.txt
```

如果仅运行基础深度模式，也可以执行：

```bash
pip install -r requirements.txt
```

### 3. 模型与资源准备

- 需确保树木分割模型权重文件位于 `models/sam/` 目录下。
- 单目三维重建模式首次运行时，可能需要下载较大的模型权重文件。
- 若本机未正确配置 CUDA，系统会自动退回 CPU 模式运行，但处理速度会明显下降。

### 4. 启动方式

深度模式启动命令：

```bash
python main.py
```

单目三维重建模式启动命令：

```bash
python main_slam3r.py
```

### 5. 接口使用说明

深度模式主要通过 `/getData` 接口接收完整输入数据。单目三维重建模式除兼容 `/getData` 外，还提供简化接口 `/getDataSimple`，只需上传图像和提示点即可完成处理。处理完成后，可通过 `/get_point_cloud` 下载点云结果文件。

### 6. 注意事项

- 建议在正式答辩前提前完成模型初始化，避免现场等待时间过长。
- 建议准备固定测试图片与测试参数，确保演示过程稳定。
- 若使用单目三维重建模式，建议搭配 GPU 运行。

## 设计思路

本作品的设计思路围绕“低成本、可扩展、可演示、可落地”的目标展开。

首先，在问题建模上，团队将树木高度测量任务拆分为若干可实现的子问题，包括树木目标提取、三维结构获取、目标点云筛选和高度计算。通过这种分层设计，系统不仅便于开发，也便于后续针对不同环节进行优化。

其次，在技术路线选择上，作品没有只依赖单一路径，而是同时保留了“基于深度数据的测量方案”和“基于单目图像的重建方案”。这样既能够利用带深度传感器设备的优势，也能够兼顾普通设备的可用性，从而提高系统的适用范围。

再次，在工程实现层面，系统采用模块化设计，将输入处理、分割、重建、点云分析、结果生成和安全校验分别组织为独立模块。这样的结构便于维护，也方便后续继续扩展前端交互、批量处理、数据统计和云端部署等能力。

最后，在展示形式上，作品注重比赛答辩的可呈现性。系统可以直接输出可视化标注结果和点云文件，便于评委理解作品流程和实际效果，也增强了作品从算法到应用的完整性。

## 设计重点难点

### 1. 树木目标区域的准确提取

树木在真实场景中常受到背景复杂、枝叶遮挡、光照变化等因素影响，导致树木目标边界不清晰。如何利用有限的提示信息准确分割出树木主体，是系统设计中的重要难点。为解决这一问题，作品采用基于提示点的目标分割，并通过二次预测与形态学处理优化掩码结果。

### 2. 图像空间与点云空间的对应关系建立

树木高度测量最终是在点云空间中完成的，但树木目标最初是在图像空间中分割出来的，因此必须解决图像掩码与点云索引之间的映射问题。尤其是在单目三维重建模式下，点云分辨率与原图尺寸存在差异，增加了映射处理的复杂度。

### 3. 点云噪声对测量结果的影响

无论是深度模式还是单目重建模式，点云中都可能包含离群点、背景点和重建误差点。如果直接基于原始点云计算高度，会显著影响结果稳定性。因此，系统在树木点云筛选后加入离群点去除和局部清洗步骤，以提高测量结果的可信度。

### 4. 单目三维重建的尺度稳定性问题

与直接使用深度传感器相比，基于单目图像的三维重建在绝对尺度上天然存在不确定性。如何在普通 RGB 图像条件下尽可能获得可用的树木高度估计结果，是作品设计中的核心难点之一。当前系统已实现基础测量流程，后续仍可通过尺度标定、参考物引入等方式进一步提升精度。

### 5. 算法效果与工程可演示性的平衡

比赛作品不仅需要体现技术深度，也需要具备较好的现场演示效果。模型初始化速度、接口稳定性、结果可视化程度和操作流程清晰度，都会影响最终展示效果。因此，本作品在算法实现之外，也对接口设计、返回结构、安全校验和结果导出进行了工程化处理，以兼顾算法能力与答辩展示需求。
