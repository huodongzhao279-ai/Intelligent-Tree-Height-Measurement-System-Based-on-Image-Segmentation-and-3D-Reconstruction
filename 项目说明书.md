## 摘要

树木高度是林业资源调查、城市绿化管理、生态环境评估和智慧农业监测中的重要指标。传统人工测量方式效率较低，且在复杂环境下存在较大的操作成本；基于专业设备的测量方式虽然精度较高，但设备成本和使用门槛也相对较高。针对上述问题，本作品设计并实现了一套基于图像分割与三维重建的树木高度智能测量系统。系统面向普通相机、手机图像或移动端采集数据，结合目标分割、点云构建、点云筛选、包围盒测量等技术，实现树木目标区域识别、三维结构重建和树木高度估计。

本系统包含两种工作模式。第一种模式基于手机深度数据，利用图像、深度图和相机内参完成点云重建，并测量树木高度；第二种模式集成单目三维重建能力，在无深度传感器条件下，根据 RGB 图像完成三维点云生成和高度估计，从而提升设备兼容性和应用范围。系统后端基于 Flask 构建接口服务，支持原始协议上传、简化表单上传、结果可视化和点云导出，并加入了上传校验、请求约束、图片格式校验和安全响应头等基础安全机制。

本作品的创新点主要体现在：面向树木测量任务的图像分割与三维重建协同流程设计；支持“有深度”和“无深度”两种测量路径；通过掩码筛选、轮廓约束、离群点去除和 AABB 包围盒分析实现树木高度估计；在工程实现层面提供了较完整的接口服务、结果导出和可扩展部署能力。该系统可应用于校园树木调查、园林养护、农林资源统计和智慧生态场景，具有一定的实际应用价值和推广潜力。

关键词：树木高度测量；图像分割；三维重建；点云处理；人工智能应用

## 1. 作品背景与意义

### 1.1 研究背景

在林业调查、园林规划、城市生态治理和农林资源监测等场景中，树木高度是一项基础且关键的参数。传统人工测量通常依赖测高仪、卷尺或人工目测，不仅效率低，而且在树木密集、地形复杂、视角受限的环境中容易产生较大误差。随着人工智能、计算机视觉和三维感知技术的发展，利用图像和智能算法对目标进行自动识别与几何估计，已经成为提升测量效率和智能化水平的重要方向。

### 1.2 问题分析

现有树木测量方案普遍存在以下问题：

- 依赖专用测量设备，成本较高，不利于大范围普及。
- 传统人工测量流程复杂，效率较低，难以满足高频采集需求。
- 部分基于深度传感器的方案受设备型号限制，适用范围较窄。
- 复杂背景、遮挡、光照变化等因素会影响树木区域提取和结果稳定性。

### 1.3 作品意义

本作品尝试以低成本视觉感知替代部分传统测量流程，使普通图像采集设备也能参与树木高度估计。该系统能够提升树木测量的自动化程度，降低设备依赖和人工成本，具有以下意义：

- 为校园、园林和林业场景提供更便捷的数字化测量方案。
- 促进人工智能技术在农林与环境监测中的落地应用。
- 为后续扩展至树冠宽度、树木体积、生长趋势分析等任务提供基础。

## 3. 总体方案设计

### 3.1 总体思路

本系统围绕“图像输入 -> 树木分割 -> 三维重建 -> 点云筛选 -> 高度计算 -> 结果输出”的核心流程展开设计。系统通过用户在图像上选取树木提示点，借助分割模型得到树木区域掩码，再结合深度图或单目三维重建模型生成场景点云；随后根据树木掩码筛选点云中的目标点，进行离群点去除和几何分析，最终利用轴对齐包围盒估算树木高度，并输出带标注的图像结果和点云文件。

### 3.2 系统架构

系统主要由以下部分组成：

- 输入处理模块：负责接收上传图像、深度数据、相机参数和点击坐标。
- 图像分割模块：根据用户提示点生成树木目标掩码。
- 三维重建模块：根据不同模式生成初始点云。
- 点云筛选与测量模块：提取树木点云并计算高度。
- 接口服务模块：对外提供结果访问与文件下载能力。
- 安全校验模块：限制上传格式、尺寸和请求行为。

### 3.3 两种工作模式

#### 3.3.1 深度模式

深度模式对应项目中的 [`main.py`](E:\tree\TreeHeight - 副本\main.py)，适用于具备深度信息的输入场景。系统在收到图像、深度数据、置信度数据和相机内参后，重建场景点云，并计算树木高度。

#### 3.3.2 单目重建模式

单目重建模式对应项目中的 [`main_slam3r.py`](E:\tree\TreeHeight - 副本\main_slam3r.py)，适用于仅有 RGB 图像的场景。该模式集成单目三维重建能力，在无深度传感器条件下完成点云生成和树木高度测量，从而提升系统的设备兼容性。

## 4. 关键技术与实现原理

### 4.1 树木目标分割

项目中的 [`seg.py`](E:\tree\TreeHeight - 副本\seg.py) 实现了基于提示点的目标分割。系统使用用户给定的两个正向提示点，对图像中的目标树木进行掩码预测。分割阶段采用两次预测流程：第一次生成多个候选掩码并比较得分，选取最优结果后再次预测，以获得更精细的分割边界。该设计有助于提升目标区域提取的准确性，减少背景误检。

在得到原始掩码后，系统进一步进行阈值二值化和形态学开运算，去除小面积噪声，并输出更稳定的树木区域掩码，为后续点云筛选提供可靠依据。

### 4.2 基于深度信息的点云构建

在深度模式下，项目中的 [`calculate.py`](E:\tree\TreeHeight - 副本\calculate.py) 会读取图像、深度数据、原始深度数据和置信度数据，并进行如下处理：

- 校验深度数据尺寸。
- 对深度图进行旋转与形状修正。
- 使用高置信度原始深度值替换基础深度值，提升深度有效性。
- 对无效数值进行清洗。
- 结合相机内参和图像颜色信息生成彩色点云。

该模式可充分利用移动设备提供的深度信息，在较低计算成本下完成场景重建。

### 4.3 基于单目图像的三维重建

在单目重建模式下，项目中的 [`calculate_slam3r.py`](E:\tree\TreeHeight - 副本\calculate_slam3r.py) 调用 [`slam3r_integration.py`](E:\tree\TreeHeight - 副本\slam3r_integration.py) 中的接口，利用单目 RGB 图像完成点云重建。该模式的特点是无需依赖手机深度传感器，只要有普通图像输入即可运行，适合设备条件受限的实际场景。

考虑到单目三维重建输出点云与原始图像尺寸存在差异，系统在树木掩码与点云对应时进行了裁剪、缩放和索引映射处理，以保证树木目标区域能够准确映射到点云空间中。

### 4.4 点云筛选与高度计算

在点云构建完成后，系统会根据树木掩码提取目标轮廓，只保留满足空间位置约束和面积约束的有效轮廓。随后，系统针对每个轮廓执行如下步骤：

- 构建当前轮廓对应的局部掩码。
- 根据局部掩码提取点云中的有效索引。
- 对目标点云执行离群点去除，减少噪声影响。
- 计算目标点云的轴对齐包围盒。
- 以包围盒在高度方向上的尺寸作为树木高度估计值。

该方法工程实现简单、可解释性较强，适合竞赛作品展示“从视觉感知到几何测量”的完整链路。

### 4.5 结果可视化与导出

系统会将树木掩码与原始图像进行透明度融合，绘制边界框和高度文字标注，生成便于展示的结果图；同时支持将筛选后的树木点云保存为 `.ply` 文件，便于后续在 Open3D 等工具中进行查看和分析。

### 4.6 接口安全与输入校验

项目中的 [`web_security.py`](E:\tree\TreeHeight - 副本\web_security.py) 对接口调用进行了基础安全增强，主要包括：

- 限制单次上传大小。
- 限制图片尺寸和像素数量。
- 校验上传文件格式。
- 校验数值范围和请求方法。
- 使用原子写入方式保存响应数据。
- 增加缓存控制与基础安全响应头。

这些设计提高了系统在实际运行过程中的稳定性和安全性。

## 5. 系统功能设计

### 5.1 功能概述

本系统实现了以下核心功能：

- 树木图像输入与参数接收。
- 树木区域智能分割。
- 场景三维点云构建。
- 目标点云筛选与树木高度计算。
- 结果图像生成与点云文件导出。
- 原始接口兼容与简化接口支持。

### 5.2 主要接口设计

#### 5.2.1 `/getData`

该接口用于接收完整输入数据。在深度模式下，系统需要上传图像、深度数据、原始深度数据、置信度数据以及相机内参与点击坐标；在单目重建模式下，系统兼容原有协议，但重点使用图像与点击坐标完成三维重建。

#### 5.2.2 `/getDataSimple`

该接口由单目重建模式提供，只需上传图像和点击坐标即可运行，更适合演示和快速测试。

#### 5.2.3 `/get_point_cloud`

该接口用于下载当前请求对应的树木点云文件，便于评委或使用者查看重建结果。

### 5.3 输出结果

系统输出内容主要包括：

- 树木高度测量结果。
- 带标注的掩码融合图像。
- 当前工作模式标识。
- 请求编号。
- 可下载的点云文件地址。

## 7. 运行环境与部署说明

### 7.1 开发环境

- 操作系统：Windows 10/11 或 Linux
- Python 版本：建议 Python 3.11
- 核心框架：Flask、PyTorch、OpenCV、Open3D、Pillow
- 硬件建议：具备 NVIDIA GPU 的设备可获得更好的运行效果

### 7.2 依赖安装

深度模式可参考 [`requirements.txt`](E:\tree\TreeHeight - 副本\requirements.txt)，单目重建模式可参考 [`requirements_slam3r.txt`](E:\tree\TreeHeight - 副本\requirements_slam3r.txt)。

建议部署步骤如下：

```bash
conda create -n treeheight_slam3r python=3.11
conda activate treeheight_slam3r
pip install torch==2.5.0 torchvision==0.20.0 --index-url https://download.pytorch.org/whl/cu118
pip install -r requirements_slam3r.txt
```

### 7.3 启动方式

深度模式启动命令：

```bash
python main.py
```

单目重建模式启动命令：

```bash
python main_slam3r.py
```

### 7.4 注意事项

- 首次运行单目重建模式时，模型权重可能需要较长时间初始化。
- 若显卡环境未配置成功，系统会退回 CPU 模式运行，速度较慢。
- 单目重建得到的点云尺度可能仍需进一步标定。
- 演示前应提前准备测试图片、提示点和输出样例，避免现场等待模型初始化。
