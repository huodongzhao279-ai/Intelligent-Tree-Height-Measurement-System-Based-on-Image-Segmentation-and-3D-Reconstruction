《编译原理》第五次作业

变量声明语句的翻译模式设计

一、题目描述

设下面文法产生变量的声明语句，分别设计基于LL(1)、LR(k)的翻译模式，实现把变量的类型和值存入到符号表中，并分析符号串 int a,b,c=5。

S → T L

L → id, L | id=V

T → int | real

V → num

第一部分：基于LL(1)的翻译模式（自上而下）

1.1 属性定义

定义以下属性：

  • T.type —— T的类型，综合属性；
  • L.type —— L的类型，继承属性（从T继承）；
  • L.val —— L中变量的值，综合属性；
  • V.val —— V的值，综合属性；
  • id.place —— id在符号表中的入口地址，综合属性，由词法分析程序给出；
  • num.val —— num的数值，综合属性，由词法分析程序给出。

1.2 翻译模式

S → T {L.type = T.type} L

    {对L中所有变量调用addtype()}


L → {id.place = &id} id , {L₁.type = L.type} L₁

    {addtype(id.place, L.type, null)}


L → {id.place = &id} id = V

    {addtype(id.place, L.type, V.val)}


T → int {T.type = int}


T → real {T.type = real}


V → num {V.val = num.val}

1.3 LL(1)分析时的语义动作编号

由于在LL(1)分析时，把语义动作代码压栈太长，因此给每一段语义代码一个编号：

S → T @1 L @2

L → @3 id , @4 L₁ @5

L → @3 id = V @6

T → int @7

T → real @8

V → num @9

语义动作定义：

  • @1: L.type = T.type
  • @2: （完成所有变量的类型和值登记）
  • @3: id.place = &id
  • @4: L₁.type = L.type; addtype(id.place, L.type, null)
  • @5: （继续处理后续变量）
  • @6: addtype(id.place, L.type, V.val)
  • @7: T.type = int
  • @8: T.type = real
  • @9: V.val = num.val

1.4 LL(1)分析表

非终结符 int real id , = num $
S S→T @1 L @2 S→T @1 L @2
T T→int @7 T→real @8
L L→@3 id , @4 L₁ @5
或 L→@3 id = V @6
V V→num @9

注：L的产生式需要向前看两个符号来区分（看id后面是","还是"="）。

1.5 分析符号串 int a,b,c=5 的过程

语法树（带属性标注）：

S / \ T L L.type=int | /|\ int id , L L.type=int, addtype(&a, int, null) (a) /|\ id , L L.type=int, addtype(&b, int, null) (b) /|\ id = V addtype(&c, int, 5) (c) | num (5)

第二部分：基于LR(k)的翻译模式（自下而上）

2.1 属性文法

为了在构造识别活前缀的DFA时具有唯一的初态，把G[S]改造为G[S']，并对产生式进行编号：

⓪ S' → S

① S → T L

② L → id , L

③ L → id = V

④ T → int

⑤ T → real

⑥ V → num

为S、T、L、V、id、num设置综合属性：

  • T.type —— 类型（int或real）
  • L.list —— 变量列表（包含变量名和值）
  • V.val —— 数值
  • id.name —— 变量名
  • num.val —— 数值

2.2 语义规则

S' → S

    { print(符号表); }


S → T L

    { for each (name, val) in L.list:

        addtype(name, T.type, val); }


L → id , L₁

    { L.list = [(id.name, null)] + L₁.list; }


L → id = V

    { L.list = [(id.name, V.val)]; }


T → int

    { T.type = int; }


T → real

    { T.type = real; }


V → num

    { V.val = num.val; }

第三部分：LL(1)与LR(k)翻译模式的比较

比较项 LL(1)（自上而下） LR(k)（自下而上）
分析方向 自顶向下，最左推导 自底向上，最左规约
属性传递 需要继承属性（L.type从T传递） 主要使用综合属性
类型传递方式 T.type通过继承属性传给L L收集变量列表，S规约时统一设置类型
符号表填写时机 遇到每个变量时立即填写 规约到S时批量填写
实现复杂度 需要处理继承属性在栈中的传递 使用综合属性，实现相对简单

结论

通过上述两种翻译模式，成功分析了符号串 int a,b,c=5，并将变量a、b、c的类型（int）和值（a、b为null，c为5）正确地存入了符号表中。

  • LL(1)翻译模式采用自上而下的分析方法，需要使用继承属性来传递类型信息；
  • LR(k)翻译模式采用自下而上的分析方法，主要使用综合属性，在规约时收集变量信息并统一处理类型。
